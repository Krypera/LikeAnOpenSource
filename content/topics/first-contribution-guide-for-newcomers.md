# Create a first contribution guide for newcomers

This topic should become a step-by-step guide for people who want to contribute to open source but do not yet know how to begin.

## What the guide should cover

- how to choose a realistic first repository
- what to look for in contribution docs and issue labels
- how to decide whether a task is still active
- how to ask for context without wasting a maintainer's time
- what a "small but useful" first change looks like

## What the reader should leave with

By the end, the reader should know:

- how to choose where to start
- how to reduce the risk of picking the wrong first task
- what to do before they open a pull request

## What to avoid

- broad motivation without process
- implying that every repository is welcoming
- advice that assumes the reader already knows contributor norms
